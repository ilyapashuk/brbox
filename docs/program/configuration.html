расположение файлов конфигурации

все конфигурационные файлы должны находиться в каталоге, называемом каталогом конфигурации.

поумолчанию файлы ищутся в каталоге config рядом с исполняемым файлом приложения, но это можно переопределить, задав переменную среды BRBOX_CONFIG_DIR.

к этой переменной также всегда можно обратиться из любого конфига под именем $BRBOX_CONFIG_DIR

конфигурация и переменные среды

в данный момент для хранения значений конфигурации используется хранилище переменных среды процесса приложения. это значит, что если какое-то значение не задано в конфигурационном файле, но задано в переменных среды, то будет взято значение из среды. однако, если значение задано в конфиге, оно перекроет значение из переменных среды.

главный конфиг

этот файл находится под именем config.txt в каталоге конфигурации. это должен быть utf8 с bom.

файл построчно считывается при каждом старте приложения независимо от того, какая из подкоманд была вызвана.

строки, начинающиеся с символа "#" являются комментариями

каждая строка должна содержать одну опцию, имя опции от значения отделяется пробелом. ситуации с пробелами внутри значения обрабатываются в bash-подобном стиле, то есть с использованием одинарных и двойных ковычек и экранированием некоторых символов при помощи знака обратной косой черты.

допускается, также в bash-подобном стиле, обращение к переменным, то есть к уже заданным выше значениям конфига и переменным среды, первые имеют приоритет над вторыми.

имеющиеся опции

ниже дан список имеющихся опций конфига.

  • BRBOX_LINE_SIZE: указывает количество символов, умещающихся в одной строке брайлевского листа. обычно это значение следует устанавливать равным тому, которое задано в используемом брайлевском принтере.
  • BRBOX_BXT_TABLE: указывает путь к файлу таблицы, используемой для кодировки в формате bxt. используется командами prepair и conv.
  • BRBOX_LANG_LETTERS: список всех букв рабочего языка, в данном случае русского, в нижнем регистре. используется подсистемой переноса слов, чтобы определить, какие слова можно прогонять через библиотеку расставления переносов. перенос многоязыких текстов пока не поддерживается.
  • BRBOX_HYPHENATION_PATTERNS_FILE: указывает путь к файлу с паттернами переноса слов для команды prepair. паттерны можно скачать с ctan или отсюда. файл должен быть в utf8 без bom.
  • BRBOX_HYPHENATION_EXCEPTIONS_FILE: содержит список слов, по слову на строку, с отмеченными символом дефиса местами переноса, например, "таб-ли-ца." в этот файл можно внести редкие слова, не подчиняющиеся стандартным правилам переноса.
